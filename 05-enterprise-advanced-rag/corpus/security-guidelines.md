# Security Guidelines

## Access Control
- Use strong, unique passwords for all accounts (minimum 12 characters)
- Enable two-factor authentication (2FA) on all work accounts
- Never share passwords or credentials
- Lock your workstation when leaving your desk

## Data Protection
- Encrypt all sensitive data at rest and in transit
- Follow the principle of least privilege for data access
- Do not store customer data locally; use approved cloud storage
- Classify data according to the data classification policy

## Secure Development
- Conduct security reviews for all new features
- Use parameterized queries to prevent SQL injection
- Validate and sanitize all user inputs
- Keep dependencies up to date and scan for vulnerabilities
- Never commit secrets, API keys, or credentials to version control

## Incident Response
Report suspected security incidents immediately to security@techcorp.example. Do not attempt to investigate on your own.

## Compliance
All engineering staff must complete annual security training and acknowledge the Acceptable Use Policy.
